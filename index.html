TrajCakeWars

A fully-featured, production-ready Cake Wars 1v1 minigame engine for Spigot/Paper servers. Built for performance, customization, and seamless integration.

🏟️

Multi-Arena

Run unlimited concurrent games. Each match runs in an isolated temporary world cloned from a template.

⚙️

Fully Configurable

Every shop item, reward, message, and game parameter is configurable from a single config.yml file.

🔌

Plugin Agnostic

Works with any NPC plugin (Citizens, FancyNpcs, ZNPCsPlus), any economy plugin, any permissions plugin.

🔒

Private Sessions

Players can create invite-only matches with a unique 6-character code. Challenge your friends.

Installation

1

Drop the JAR

Place TrajCakeWars-2.1.0.jar into your server's plugins/ directory.

2

Start the server

The plugin generates its default config.yml and arenas.yml on first launch.

3

Create your template

Build your CakeWars map, then copy the world folder to cw_templates/.

4

Configure your arena

Use in-game setup commands to define spawn points, cake locations, and generators.

Quick Start

Get a working CakeWars game in under 5 minutes:

# On your server machine — stop the server first
mkdir -p cw_templates
cp -r my_cakewars_map cw_templates/desert

# Start the server, then in-game:
/cw create desert
/cw set desert spawn1        # Stand at player 1 spawn
/cw set desert spawn2        # Stand at player 2 spawn
/cw set desert cake1         # Stand on cake block 1
/cw set desert cake2         # Stand on cake block 2
/cw set desert gen1          # Iron generator island 1
/cw set desert gen2          # Iron generator island 2
/cw set desert diamond       # Diamond generator (repeat for each)
/cw set desert lobby         # Arena waiting lobby point
/cw settemplate desert desert
/cw setname desert §6§lDesert Arena
/cw enable desert
/cw save
Tip: You can repeat the /cw set <arena> diamond command at different locations to create multiple diamond generators.

Creating a Map Template

A template is a complete Minecraft world folder stored in cw_templates/ at your server root. Each game clones this template into a temporary world, ensuring the original is never modified.

Method 1: Manual Copy

# Build your map in a dedicated world, then:
server/
├── cw_templates/
│   └── desert/          ← full world folder
│       ├── region/
│       ├── level.dat
│       └── ...

Method 2: In-Game Command

Stand in the world containing your map and run:

/cw savetemplate desert

This copies the current world to cw_templates/desert/.

Important: The template world should contain your complete CakeWars map including shop NPCs (if using entity-based NPCs). All entities within the world are cloned alongside the terrain.

Shop NPCs

The plugin does not spawn shop NPCs itself. Instead, it detects interactions with existing entities based on configurable name keywords. This makes it compatible with any NPC plugin or even vanilla renamed mobs.

How It Works

  1. Place NPCs in your template world using any method (Citizens, FancyNpcs, name tags, etc.)
  2. Give them names containing configured keywords
  3. When a player right-clicks the NPC in a game, the corresponding shop opens

Default Keywords

ShopTriggers when name contains
Items ShopItems, Item, Combat, Armes
Blocks ShopBlocks, Blocs, Build, Construction
Keyword matching is case-insensitive and substring-based. A villager named "§e§lItems Shop" will trigger the items shop because it contains "Items". Color codes are stripped before matching.

Resource Generators

Generators automatically drop resource items at configured locations during a game.

TypeResourceDefault IntervalMax on Ground
Iron (per island)Iron Ingot2 seconds (40 ticks)32
Diamond (shared)Diamond30 seconds (600 ticks)4

Generators have a ground item cap — they stop spawning when the configured maximum is reached within a 3-block radius. This prevents item accumulation and lag.

Death System

TrajCakeWars uses a fake death system. Players never see the vanilla death screen. When a player takes fatal damage:

  • The damage is cancelled and health is reset
  • The player is set to Spectator mode at the arena lobby
  • A countdown title appears on screen (configurable duration)
  • After countdown, the player respawns at their island with a starter kit
  • If their cake has been destroyed, they are permanently eliminated

Freeze Phase

During the 5-second countdown before a game starts, players are frozen in place. They can look around but cannot move, break blocks, or take damage. Movement is enforced via repeated teleportation.

TNT & Explosions

Explosions (TNT, fireballs) only destroy blocks placed by players during the game. The original map terrain is always protected and never modified.

Matchmaking

The matchmaking system supports three queue types that work together:

Queue TypeHow to JoinBehavior
Public (Random)GUI → Play, or /cw joinMatches with any player, picks any available arena
Map-SpecificGUI → Choose Map → Click arenaOnly matches with players queued for the same map
PrivateGUI → Private, or /cw private <arena>Creates a session with a 6-char code

The matchmaker runs every second and also attempts cross-queue matching: a random-queue player can be matched with a map-specific player if the requested map is available.

Private Sessions

Private sessions allow players to create invite-only matches:

# Host creates a session
/cw private desert
# → "Session created! Code: K4HN7P"

# Friend joins with the code
/cw join K4HN7P
# → Game starts immediately

Sessions can also be browsed through the GUI. Open sessions are listed with host name, map, and a join button.

Cake Mechanic

The cake is the core objective. Each player has a cake block on their island.

PropertyValue
Total bites to destroy7 (vanilla cake behavior)
Cooldown between bites1 second
InteractionRight-click only (left-click is blocked)
Self-eat protectionPlayers cannot eat their own cake

When a cake is fully eaten, the owning player can no longer respawn. Their next death results in permanent elimination and the opponent wins.

Why right-click? The cake must be eaten (right-click), not broken (left-click). This is intentional — it adds a risk/reward dynamic where the attacker must stand still and eat, leaving them vulnerable.

Shops

Shops are fully driven by config.yml. There are two default shops: Items Shop and Blocks Shop. Every item can be customized.

Item Types

TypeBehavior
swordAuto-replaces any lower-tier sword in the player's inventory. Always placed in slot 0 (main hand).
armor-setAuto-equips chestplate + leggings. Replaces lower-tier armor. Requires chest and legs fields.
armor-pieceAuto-equips a single armor piece. Replaces lower-tier in the same slot.
normalAdds the item to inventory. Supports amount field for stacks.

Armor Tier Hierarchy

Leather (0) → Chainmail (1) → Iron (2) → Diamond (3)

A player can only equip armor of equal or higher tier. Purchasing a higher tier automatically replaces the lower one.

Configuration — General

settings:
  prefix: "&d&lCakeWars &r&7» &r"   # Chat message prefix
  lobby-world: "world"               # Main lobby world name

game:
  countdown: 5            # Pre-game freeze duration (seconds)
  respawn-time: 5         # Respawn countdown when cake is alive
  max-duration: 600       # Max game length in seconds (0 = infinite)

Configuration — Generators

generators:
  iron:
    interval: 40     # Ticks between spawns (20 ticks = 1 second)
    max: 32          # Max items on ground per generator
  diamond:
    interval: 600
    max: 4

Configuration — Rewards

rewards:
  enabled: true
  winner:
    commands:
      - "give %winner% diamond 5"
      - "eco give %winner% 100"       # Works with Vault/Essentials
      - "broadcast &6%winner% &7won CakeWars against &c%loser%&7!"
    message: "&6⭐ VICTORY! &a+5 Diamonds +100 coins"
  loser:
    commands: []
    message: "&cDefeat. Try again!"

Available Placeholders

PlaceholderDescription
%winner%Username of the winning player
%loser%Username of the losing player
%arena%Display name of the arena

Configuration — Shop Items

Both shops are fully defined in config.yml. Here is the structure for a single item:

items-shop:
  title: "&d&lItems Shop"
  size: 45                     # Inventory size: 9, 18, 27, 36, 45, 54
  items:
    my-custom-item:            # Unique key (any name)
      slot: 10                  # Inventory slot position
      type: normal              # sword | armor-set | armor-piece | normal
      material: GOLDEN_APPLE    # Bukkit Material name
      name: "&6Golden Apple"     # Display name (supports color codes)
      amount: 1                 # How many given per purchase
      price: 2                  # Cost
      currency: DIAMOND         # IRON_INGOT or DIAMOND
      lore:                     # Optional description lines
        - "&7Heals 2 hearts"
        - "&7Gives absorption"

For armor-set type, use chest and legs instead of material:

    iron-set:
      slot: 15
      type: armor-set
      chest: IRON_CHESTPLATE
      legs: IRON_LEGGINGS
      name: "&fIron Set"
      price: 6
      currency: DIAMOND

Configuration — NPC Detection

shop-detection:
  items-keywords:      # Entity names containing these open the Items Shop
    - "Items"
    - "Item"
    - "Combat"
    - "Weapons"
  blocks-keywords:     # Entity names containing these open the Blocks Shop
    - "Blocks"
    - "Build"
    - "Construction"

Matching is case-insensitive and substring-based. Color codes are stripped before comparison. Works with any living entity (Villagers, Players, Zombies — whatever your NPC plugin creates).

Technical Architecture

World Lifecycle

cw_templates/desert/     ← immutable template (never modified)
         │
         │  cp -r (file copy)
         ▼
cw_game_a8f3c2d1/        ← temporary clone (auto-generated name)
         │
         │  WorldCreator.createWorld()
         ▼
Loaded Bukkit World       ← game runs here
         │
         │  Game ends
         ▼
Bukkit.unloadWorld()      ← world unloaded from memory
         │
         │  Recursive file delete
         ▼
(deleted)                 ← no trace left

FAQ

Can I add more than 2 shops?

Currently the plugin supports two shops: Items and Blocks, each triggered by configurable keywords. Adding more shop categories requires code modification. However, you can fit up to 54 items per shop, which is usually sufficient.

How many concurrent games can run?

Limited only by server RAM and disk speed. Each game world is typically 5-50MB depending on map size. With 4GB RAM, expect 10-20 concurrent games comfortably.

Does TNT damage the map?

No. Explosions only destroy blocks placed by players during the game. The original template terrain is always protected.

What happens if the server crashes during a game?

Temporary game worlds (cw_game_*) will remain on disk. They are harmless but take up space. You can safely delete any cw_game_* folders when the server is stopped.

Can I use this for 2v2 or 4v4?

The current version is designed for 1v1. Team support would require code modifications to the matchmaking and game instance systems.

How do I update the plugin?

Replace the JAR file and restart the server. Your config.yml and arenas.yml are preserved. New config options will use their defaults until you add them manually.
